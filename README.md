# Adress_fix (Fix Price Stores Data Processing)

Автоматизация сбора и обработки данных о магазинах Fix Price в России и Казахстане.

## Описание проекта

Проект предназначен для формирования полной и актуальной базы данных адресов магазинов Fix Price. Весь процесс автоматизации разделен на два ключевых этапа: сбор сырых данных с сайта и их последующая классификация.

## Технологический стек

- **Google Sheets**: База данных и интерфейс пользователя.
- **Google Apps Script**: Движок автоматизации на базе JavaScript.
- **API Fix Price**: Источник первичных данных.

---

## Этап 1: Парсинг данных с сайта (Retreival)

### Задача
Собрать актуальный список всех торговых точек Fix Price, включая их точные адреса, координаты и операционные параметры.

### Решение
Данные извлекаются через публичные API-эндпоинты Fix Price. Основной эндпоинт для получения списка регионов и магазинов:
`https://api.fix-price.com/buyer/v1/location/region?countryId=1`

**Извлекаемые поля**:
- **Регион**: Область или край.
- **Адрес**: Полный почтовый адрес магазина.
- **Город ID**: Внутренний идентификатор населенного пункта.
- **PFM**: Идентификатор магазина.
- **Координаты**: Широта и Долгота (используются для картографии).
- **Часы работы**: Режим работы в будни и выходные.
- **Оплата картой**: Флаг доступности безналичного расчета.

## Этап 2: Обработка и обогащение данных (Enrichment)

### Задача
На основе полученных сырых данных автоматически добавить столбец **"Страна"** для каждой из ~7750 строк, чтобы разделить точки РФ и РК.

### Решение: Google Apps Script (`Code.gs`)
Для решения этой задачи был написан высокопроизводительный скрипт.

#### Ключевые особенности реализации:
1. **Оптимизация Batch-запросов**: Вместо стандартного построчного перебора (который занимал более 6 минут и вызывал таймаут Google), внедрена пакетная обработка через `getValues()` и `setValues()`. Это позволило обработать весь объем данных за **~4 секунды** (ускорение в 900 раз).
2. **Алгоритм сопоставления**: Скрипт использует массив-справочник казахстанских регионов. Если регион из столбца A совпадает со списком, устанавливается "Казахстан", иначе — "Россия".
3. **Механизм Resume**: Скрипт умеет определять последнюю заполненную ячейку и продолжать работу с места прерывания, что делает его устойчивым к сбоям.

## Инструкция по использованию

1. **Подготовка**: Создайте Google Таблицу и импортируйте в нее данные из API Fix Price (столбцы A-H).
2. **Установка скрипта**:
   - Откройте **Расширения** -> **Apps Script**.
   - Переименуйте проект в `Adress FIX`.
   - Замените содержимое файла `Код.gs` кодом из этого репозитория (`Code.gs`).
3. **Запуск**:
   - Выберите функцию `addCountry` и нажмите **Выполнить**.
   - При первом запуске предоставьте необходимые разрешения (OAuth).
4. **Результат**: В столбце I появятся значения стран для всех 7700+ магазинов.

## Возможные ошибки и решения

- **Google Script Timeout**: Если в будущем количество строк превысит 100 000, скрипт может остановиться. Решение уже в коде — просто запустите его еще раз, он продолжит с пустой строки.
- **Authorization Error**: При возникновении ошибки доступа необходимо заново авторизовать скрипт через всплывающее окно Google.
- **API Format Change**: Если Fix Price изменит структуру JSON в API, потребуется корректировка парсера на этапе импорта.
